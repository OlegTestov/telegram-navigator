# telegram-navigator

**AI-навигатор по Telegram-каналам — авто-индексация, классификация по темам, умный поиск и дайджесты.**

Бот автоматически собирает посты из Telegram-каналов, раскладывает их по темам с помощью Google Gemini, формирует удобное оглавление и присылает подписчикам регулярные сводки новых публикаций. Интерфейс на русском и английском — язык определяется автоматически.

[🇬🇧 English](README.md)

## Возможности

- **Авто-индексация** — читает историю канала через Telethon и сохраняет посты в базу
- **Классификация** — Google Gemini раскладывает посты по темам и добавляет описания
- **Оглавление** — формирует компактное интерактивное оглавление для каждого канала
- **Умный поиск** — гибридный поиск (ключевые слова + семантика) по конкретному каналу или по всем сразу. Семантика — при подключении OpenAI embeddings
- **Дайджест** — подписчики получают сводку новых постов каждые 3 часа
- **Два языка** — русский / английский интерфейс, определяется из настроек Telegram или переключается вручную
- **Выбор базы данных** — SQLite (работает из коробки) или Supabase (PostgreSQL + pgvector)

## Сценарии использования

- **Следить за множеством каналов** — добавьте все интересные вам каналы и каждые 3 часа получайте краткую сводку. Заинтересовал пост — кликните и читайте оригинал.
- **Изучить новый канал** — добавили интересный канал? Бот составит оглавление с самыми популярными, полезными и актуальными постами, разбитыми по темам — вместо прокрутки сотен сообщений.
- **Собрать базу знаний** — проиндексируйте подборку каналов и получите постоянно пополняемую базу с поиском по любой теме — AI, крипто, инструменты разработки и что угодно ещё.
- **Поиск и исследования** — нужно найти тот самый пост про конкретный инструмент? Ищите внутри одного канала или сразу по всем — гибридный поиск учитывает и ключевые слова, и смысл запроса.

## Быстрый старт (Docker)

```bash
git clone https://github.com/OlegTestov/telegram-navigator.git
cd telegram-navigator
cp .env.example .env
# Заполните .env (см. Требования ниже)
docker compose up -d bot
```

## Быстрый старт (локально)

```bash
git clone https://github.com/OlegTestov/telegram-navigator.git
cd telegram-navigator
make setup          # создаёт venv, устанавливает зависимости, копирует .env.example
# Заполните .env
make run            # запускает бота
```

## Первый запуск

Когда бот запущен:

1. **Откройте бота** в Telegram и отправьте `/start`
2. **Добавьте канал** — нажмите "Добавить канал" и отправьте ссылку вида `t.me/channelname`
3. **Запустите планировщик** — он загрузит посты, классифицирует их и подготовит оглавление:
   ```bash
   # Docker
   docker compose run --rm scheduler
   # Локально
   make scheduler
   ```
4. **Посмотрите результат** — вернитесь в бота, выберите канал, нажмите "Создать оглавление"
5. **Настройте автозапуск**, чтобы новые посты подхватывались каждый час:
   ```
   # Добавьте в crontab (crontab -e)
   0 * * * * cd /path/to/telegram-navigator && docker compose run --rm scheduler >> data/scheduler.log 2>&1
   ```

## Требования

| Параметр | Как получить |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Создайте бота через [@BotFather](https://t.me/BotFather) |
| `TELEGRAM_API_ID` / `TELEGRAM_API_HASH` | Зарегистрируйтесь на [my.telegram.org](https://my.telegram.org) |
| `TELEGRAM_SESSION_STRING` | Запустите `make session` локально (нужен номер телефона и код из Telegram) |
| `GEMINI_API_KEY` | [Google AI Studio](https://aistudio.google.com/apikey) |
| `ADMIN_TELEGRAM_ID` | Отправьте `/start` боту [@userinfobot](https://t.me/userinfobot) |

> **Важно:** `generate_session.py` нужно запускать локально (не в Docker) — скрипт запрашивает номер телефона и код подтверждения в интерактивном режиме.

## Архитектура

```
telegram-navigator/
├── src/
│   ├── main.py              # Точка входа бота (polling)
│   ├── scheduler_main.py    # Планировщик: загрузка → классификация → скоринг → оглавление → дайджест
│   ├── bot/                  # Обработчики команд, кнопок, клавиатуры, тексты интерфейса
│   ├── config/               # Настройки, константы, промпты для LLM
│   ├── database/             # Фабрика БД, модели, реализации для SQLite и Supabase
│   ├── services/             # Загрузчик, классификатор, скоринг, генератор оглавлений, дайджест, embeddings
│   └── utils/                # Вспомогательные функции, i18n, типы ошибок
├── Dockerfile
├── docker-compose.yml
├── Makefile
└── schema.sql                # Схема для Supabase (SQLite создаёт таблицы сам)
```

**Два процесса:**
1. **Бот** (`src.main`) — принимает команды и нажатия кнопок через Telegram polling
2. **Планировщик** (`src.scheduler_main`) — запускается по расписанию: загружает новые посты → классифицирует → обновляет оглавление → рассылает дайджесты

## База данных

Бот поддерживает две базы данных. Выбор — через `DB_BACKEND` в `.env`.

### SQLite (по умолчанию)

Работает из коробки — база создаётся автоматически при первом запуске.

```env
DB_BACKEND=sqlite
SQLITE_DB_PATH=data/content_table.db
```

Для гибридного поиска нужно расширение `sqlite-vec` (уже включено в `requirements.txt`). Без него работает только поиск по ключевым словам.

### Supabase (PostgreSQL)

Для продакшена и облачного размещения с надёжным хранилищем.

1. Создайте проект на [supabase.com](https://supabase.com)
2. Откройте **SQL Editor** и выполните [`schema.sql`](schema.sql)
3. Укажите переменные окружения:

```env
DB_BACKEND=supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-service-role-key
```

Для гибридного поиска включите расширение `vector` в настройках проекта (Dashboard → Database → Extensions).
